# Pet Nutrition Blog Automation Guide

이 저장소의 자동 생성 글은 반려동물 보호자를 위한 일반 영양 정보 콘텐츠로 작성한다.

## Project Rule Set

- 새 글 작성, 이미지, 내부 링크, SEO, 의료 표현, 중복 방지, Astro 배포 흐름은 `.codex/instructions.md`와 `rules/`, `templates/`, `checklists/`, `references/`, `taxonomy/`, `workflow/` 폴더의 지침을 우선 참고한다.
- 사용자가 별도로 요청하지 않는 한 최종 글은 게시 가능한 Astro Markdown 형식으로 작성하고, 최종 제목 1개, 설명 1개, slug 1개만 사용한다.
- 반려동물 영양 관련 글의 날짜는 업로드일이 아니라 작성일 기준으로 입력한다.

## Animal Type Rules

- 제목에 `강아지`, `개`, `반려견`, `dog`, `puppy`가 포함되면 강아지 전용 글로 작성한다.
- 제목에 `고양이`, `반려묘`, `냥이`, `cat`, `kitten`이 포함되면 고양이 전용 글로 작성한다.
- 제목에 강아지와 고양이가 모두 포함되거나 `비교`, `차이`, `공통`, `둘 다`, `함께` 같은 비교형 표현이 있으면 비교 글로 작성한다.
- 강아지 전용 글에는 고양이 전용 섹션을 넣지 않는다.
- 고양이 전용 글에는 강아지 전용 섹션을 넣지 않는다.
- 비교 글일 때만 강아지와 고양이를 구분해서 작성한다.

## Health Content Safety Rules

- 특정 질병을 진단하거나 치료를 단정하지 않는다.
- `무조건 좋다`, `완치된다`, `반드시 먹여야 한다`, `이것만 먹이면 된다` 같은 표현을 쓰지 않는다.
- 위험 증상, 질병 가능성, 장기적인 식단 변경이 필요한 경우 수의사 상담을 권장한다.
- 일반 보호자가 이해하기 쉬운 문장으로 작성한다.
- 어려운 수의학 용어는 쉬운 말로 풀어서 설명한다.
- SEO를 고려하되 키워드를 억지로 반복하지 않는다.
- 구글 애드센스 승인을 고려해 얕은 글이 아니라 정보성 있는 글로 작성한다.
- 글 마지막에는 일반 정보 제공용 콘텐츠이며 수의사 진료를 대체하지 않는다는 취지의 문장을 자연스럽게 포함한다.

## Output Expectations

- 본문은 최소 2,000자 이상 작성한다.
- 제목과 동물 타입에 맞는 내용만 다룬다.
- Markdown 이미지 문법으로 3~5개의 이미지 삽입 위치를 넣는다.
- 본문 하단에는 게시에 필요한 내용만 포함한다. 후속 글 주제 추천은 게시 본문에 넣지 않고 작성자에게 별도로 제공한다.
