# 작업 흐름 규칙

## 새 글 작성 워크플로

사용자가 주제를 입력하면 다음 순서로 작업한다.

1. 프로젝트 구조 확인
2. 기존 글 목록 확인
3. 대상 동물 판별
4. 음식/주제 유형 판별
5. 기존 글과 차별점 5개 작성
6. 최종 SEO 제목 1개 생성
7. slug 1개 생성
8. frontmatter 작성
9. 본문 작성
10. 이미지 3~4개 삽입 위치 작성
11. 이미지 생성 프롬프트 작성
12. 내부 링크 후보 작성
13. 후속 글 후보 작성
14. SEO 자가검토(내부 검토용, 본문 제외)
15. `npm run build` 실행 가능하면 실행
16. 결과 요약

## 사용자가 제목만 입력한 경우

제목을 그대로 쓰지 말고 검색 의도를 살려 SEO 제목으로 다듬는다.

예:

입력: `강아지 참외`
출력 제목: `강아지 참외 먹어도 될까? 급여 방법과 주의사항 총정리`

## 기존 글 검토 워크플로

사용자가 URL이나 파일을 주면 다음을 검토한다.

1. 제목과 메타 설명
2. H1/H2/H3 구조
3. 정보 정확성
4. 의료 표현 안전성
5. 이미지 alt
6. 내부 링크
7. FAQ
8. 참고자료
9. 애드센스 적합성
10. 개선 우선순위

## 이미지 생성 워크플로

1. 각 이미지의 목적 정의
2. 파일명 생성
3. alt 작성
4. 영어 이미지 프롬프트 작성
5. 이미지 경로를 본문에 삽입
6. 실제 이미지 파일이 없으면 TODO로 표시

## 빌드/배포 워크플로

Astro + GitHub + Cloudflare Pages 기준.

1. `npm install`이 필요한지 확인
2. `npm run build` 실행
3. 오류 수정
4. 사용자가 원하면 git commit/push 안내 또는 실행
5. Cloudflare Pages 배포는 GitHub push 후 자동

## 금지

- 사용자 허락 없이 git push 실행 금지
- 사용자 허락 없이 기존 글 대량 삭제 금지
- 사용자 허락 없이 Cloudflare 설정 변경 금지
- 빌드 오류를 무시하고 완료했다고 말하지 않기
